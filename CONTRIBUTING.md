# Contributing to whohas

Thank you for your interest in contributing!

## Reporting Bugs

Please open a [GitHub Issue](../../issues) with a clear description of the problem,
steps to reproduce, and the expected vs. actual behavior.

## Suggesting Features

Feature ideas are welcome — open a [GitHub Issue](../../issues) and describe
the use case you have in mind. This lets us discuss the approach before any
code is written.

## Code Contributions

- **Small fixes** (typos, one-liner bug fixes): feel free to open a Pull Request
  directly.
- **Non-trivial changes** (new features, refactors, API changes): please open an
  Issue first so we can agree on the direction before you invest significant
  effort.

All Pull Requests should include a short description of *what* changed and *why*.
